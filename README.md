---

## 📌 Sobre o Projeto

Este projeto foi desenvolvido como parte do **Projeto Integrado Multidisciplinar III (PIM III)** do curso de **Análise e Desenvolvimento de Sistemas** da UNIP.

O objetivo é a criação de um **Sistema Web Integrado** para uma empresa fictícia chamada **ModaMax Distribuidora e Varejo LTDA**, que atua no segmento de comércio de roupas nos modelos:

- 🛍️ **B2C (Varejo)** – Venda direta ao consumidor final  
- 📦 **B2B (Atacado)** – Venda para lojistas e revendedores  

O sistema contempla desde a análise do negócio até a implementação técnica, integrando conceitos de:

- Engenharia de Software Ágil  
- Modelagem de Banco de Dados (Relacional e/ou NoSQL)  
- Programação Orientada a Objetos (C#)  
- Desenvolvimento Web Responsivo  
- UX/UI Design  
- Análise de Dados e Machine Learning  
- Comunicação Organizacional, Liderança e Negociação  
- Acessibilidade (LIBRAS)

---

## 🎯 Objetivo Geral

Desenvolver um sistema web completo que atenda às necessidades operacionais da empresa fictícia, integrando aspectos técnicos, organizacionais e humanos em um contexto empresarial simulado, porém realista.

---

## 🏬 Sobre a Empresa Fictícia

**Nome:** ModaMax Distribuidora e Varejo LTDA  
**Segmento:** Comércio de roupas  

**Público-alvo:**
- Consumidores finais (varejo)
- Lojistas e revendedores (atacado)

**Principais Necessidades:**
- Controle de estoque integrado
- Gestão de pedidos B2C e B2B
- Cadastro e gestão de clientes
- Relatórios de vendas e desempenho
- Indicadores estratégicos baseados em dados
- Plataforma acessível e responsiva

## 🧠 Metodologia Aplicada

O projeto está estruturado nas seguintes etapas:

### 1️⃣ Definição do Negócio  
Caracterização da empresa fictícia e identificação do problema organizacional.

### 2️⃣ Engenharia de Software Ágil  
- Identificação de stakeholders  
- Levantamento de requisitos funcionais e não funcionais  
- Product Backlog  
- Priorização  
- Definição de iterações  

### 3️⃣ Modelagem de Banco de Dados  
- Modelo conceitual  
- Modelo lógico  
- Definição de tabelas e relacionamentos  
- Justificativa do uso de banco relacional e/ou NoSQL  

### 4️⃣ Desenvolvimento com C# (POO)  
- Arquitetura da aplicação  
- Implementação orientada a objetos  
- Encapsulamento, herança e polimorfismo  

### 5️⃣ Desenvolvimento Web Responsivo  
- Interface adaptável (mobile-first)  
- Integração front-end e back-end  
- Compatibilidade com múltiplos dispositivos  

### 6️⃣ UX e UI Design  
- Criação de personas  
- Definição de fluxos de navegação  
- Wireframes  
- Avaliação de usabilidade  

### 7️⃣ Machine Learning e Análise de Dados  
- Geração de relatórios  
- Indicadores de desempenho  
- Análises preditivas simples  

### 8️⃣ Comunicação, Liderança e LIBRAS  
- Estratégias de comunicação da equipe  
- Simulação de negociações  
- Propostas de acessibilidade digital  

### 9️⃣ Integração e Avaliação Final  
- Consolidação do sistema  
- Avaliação dos resultados  
- Propostas de melhorias futuras
